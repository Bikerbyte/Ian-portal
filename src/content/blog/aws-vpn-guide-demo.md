## 這篇在做什麼

做法是用 AWS EC2 開一台虛擬機器，直接選擇含有 OpenVPN 的映像，完成初始化之後，再讓自己的電腦透過 OpenVPN 連回去。

這種方式比較像是自己架一個固定節點，而不是購買商用 VPN 服務。

## 流程整理

### 1. 建立 EC2

先登入 AWS，切到你想要的區域，然後建立一台 EC2。

這一步最重要的是：

- 區域要選對
- 映像要選 OpenVPN 相關的 Marketplace AMI
- 執行個體類型要留意免費方案限制

### 2. 建立金鑰對

建立 VM 的過程中，AWS 會要求你建立或選擇登入金鑰。

這個金鑰檔之後要拿來 SSH 進機器，所以一定要保存，不然後面會很麻煩。

### 3. SSH 進去做初始化

VM 啟動後，可以在 EC2 後台找到 SSH 連線指令。  
第一次先用系統提供的方式登入，接著依映像的提示切換到正確的帳號，再設定 OpenVPN 之後會用到的密碼。

### 4. 設定 OpenVPN 管理頁

接著打開：

`https://你的公網 IP/admin`

登入後，把讓用戶端流量走 VPN 的設定打開，然後儲存並套用。

做完這一步，VPN 伺服器端就大致完成了。

### 5. 下載 Client 並連線

之後改走一般使用者入口登入，下載對應平台的 OpenVPN client。

以 Windows 來說，安裝完成後通常就能看到已經建立好的連線設定，輸入帳密後就能開始測試。

### 6. 檢查安全群組

如果連線後發現某些流量不通，很常見的原因是 EC2 的安全群組規則還沒放行。

原文裡有提到，預設只開幾個常見連接埠；如果想讓更多流量都能正常通過，通常要再調整傳入規則。

## 我會特別提醒的地方

- AWS 區域會直接影響節點位置
- 金鑰檔要留好
- 管理頁和使用者登入頁是不同入口
- 安全群組沒設好，很容易誤判成 VPN 壞掉
